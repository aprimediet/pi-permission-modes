## Tech Stack
**Language(s):** TypeScript (ESM)

**Infrastructure:** None (pure pi extension, no Docker/CI)
**Package Manager:** npm (no lock file)

## Goals
- Provide four distinct permission modes (default/plan/accept-edits/auto) that control how tool calls (edit, write, bash) are approved
- Let users cycle modes easily via Shift+Tab keyboard shortcut
- Show the current mode in the UI via a status pill and custom footer
- Persist mode and auto-follow-up depth across session restarts
- In Plan mode: restrict to read-only, produce a structured plan, then optionally execute it with [DONE:n] step tracking

## Non-Goals
- Safety guardrails
- Switching the AI model per mode
- Replacing pi's built-in --mode flag for output format
- Standalone CLI or application (pi extension only)

## Boundaries
- **Entry point:** `index.ts` (default-exported factory function)
- **Helpers:** `utils.ts` (bash allowlist, Plan extraction, DONE tracking)
- **External services:** None
- **Exposed interfaces:** Commands (`/default`, `/plan`, `/accept-edits`, `/auto`, `/mode`, `/auto-depth`), shortcuts (`Shift+Tab`, `Alt+T`), flag (`--permission-mode`), pi events (tool_call, before_agent_start, context, turn_start/end, agent_end, session_start/tree)
- **Tech constraints:** Requires pi runtime, ESM only, no third-party deps beyond pi-core peer packages
